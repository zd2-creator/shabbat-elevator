# מעלית שבת — סקר קומות (מאובטח)

טופס לרישום דיירים כדי לקבוע **באילו קומות תעצור מעלית השבת**, עם מפת בניין חיה ומעלית מונפשת.
אוסף רק **קומה + דירה + שם** — בלי ת"ז, בלי חתימה. גיליון ופרויקט חדשים.

| קובץ | יעד | תפקיד |
|------|-----|-------|
| `Code.gs` | פרויקט Apps Script **חדש** | שרת מאובטח |
| `index.html` | אירוח ציבורי (שורש הריפו) | טופס + מפת בניין |
| `admin.html` | אותו מקום | דף ועד (מוגן סיסמה) |
| `og-image.png` | אותו מקום | תמונת תצוגה לוואטסאפ |

---

## הקמה (מ‑0)

1. **גיליון Google חדש** (sheets.new). בשורה 1: `ts` | `floor` | `apt` | `name`.
2. **Extensions → Apps Script** → הדבק את `Code.gs`. שמור.
3. **Project Settings → Script Properties** → הוסף `ADMIN_PASSWORD` = סיסמה חזקה.
4. **Deploy → New deployment → Web app** · Execute as: Me · Who has access: **Anyone** → העתק כתובת `/exec`.
5. החלף `PASTE_YOUR_NEW_EXEC_URL_HERE` בשני קבצי ה‑HTML בכתובת שהעתקת.
6. העלה את שני הקבצים לאירוח (GitHub Pages / Netlify).

## התאמה אישית
בראש ה‑`<script>` בשני הקבצים:
```js
const FLOORS = 14; // מספר הקומות בבניין — שנה לפי הצורך
```

## אבטחה (מובנה)
- טופס ציבורי = **כתיבה בלבד**; `progress` מחזיר רק ספירות לפי קומה (בלי שמות).
- אדמין מוגן בסיסמה **בצד השרת** (Script Properties).
- שמות מוצגים עם `textContent` → חסין XSS.
- מינימום מידע: קומה+דירה+שם בלבד.

## איך זה נראה
- כל דייר בוחר קומה+דירה+שם ונרשם (דירה אחת = רישום אחד).
- מפת הבניין נצבעת: ככל שקומה ירוקה יותר — יותר ביקוש. קומות עם ביקוש מסומנות ב‑⏹ (שם המעלית תעצור), והמעלית המונפשת עוברת ביניהן.
- בדף הוועד: פילוח לפי קומה + רשימת נרשמים + הקומה המבוקשת ביותר — כדי להחליט על תוכנית המעלית.
